## 阶段概览

| 项目 | 内容 |
|------|------|

| **目标角色** | 数据架构师 |

## 阶段目标

本阶段是学员从"高级工程师"向"架构师"跃迁的关键阶段。结业时学员应具备以下能力：

1. **独立架构设计能力**：能独立完成大数据平台架构设计，从需求分析到技术选型到实施路线图
2. **技术选型决策能力**：能在多个技术候选方案中做合理的 Trade-Off，并清晰论证选择理由
3. **系统设计方法论**：掌握架构设计五步法，能应对各种系统设计面试题
4. **跨团队协调能力**：能与非技术利益相关者沟通技术方案，具备 Mentor 初级工程师的能力
5. **成本意识**：建立 FinOps 思维，能从成本角度审视和优化技术架构
6. **数据驱动决策**：习惯用数据和量化指标支撑技术决策

## 核心教学内容

### 1. 系统设计训练（第27-30周，共8题）

掌握架构设计五步法，通过8道真实系统设计题反复训练：
- **方法论**：需求澄清 → 容量估算 → 高层设计 → 深入设计 → 权衡分析
- **题目覆盖**：实时数仓、实时特征平台、数据联邦查询、PB级数据湖仓、离线计算平台、实时用户画像、数据质量监控、数据血缘追踪

### 2. 项目11：企业大数据平台架构设计（第31-38周，60h）

模拟为中型电商公司（日活500万，日订单100万）从零设计完整大数据平台：
- 业务需求分析与数据源盘点
- 技术架构设计与组件选型对比
- 实施路线图与人力规划
- 成本预算与 ROI 分析
- 风险识别与应对预案

### 3. 项目12：FinOps成本优化（第39-42周，20h）

对大数据平台进行系统性的成本审计与优化：
- 计算资源审计（僵尸任务、低效任务识别）
- 存储审计（冷数据、重复数据、临时数据）
- Kafka 审计（Topic 使用效率分析）
- 4大优化方案：冷热分层、弹性伸缩、任务合并、查询优化

### 4. L4 结业考核（第43-44周）

- 系统设计面试：2小时完成大数据平台设计
- 架构方案答辩：面对"CTO评审团"（3位以上评委）
- 成本优化：方案可量化节省成本 ≥30%
- Mentor 成果：带的初级学员 L2 结业通过率 ≥80%

### 5. dbt与现代数据栈（补充模块，第43-46周）

- dbt核心实战：Model/Test/Documentation/Incremental Model
- 数据测试与CI/CD：pytest+Spark测试、DVC数据版本、GitHub Actions流水线
