### 2.1 为什么需要容器化（20分钟）

#### 环境一致性问题

```
开发人员的日常噩梦：

  开发机        →  "在我电脑上能跑啊！"     ✅
  测试环境      →  "怎么报错了？"            ❌
  生产环境      →  "服务器挂了！"            💥

根本原因：
  - 操作系统版本不同（Ubuntu 20.04 vs CentOS 7）
  - Python版本不同（3.9 vs 3.7）
  - 依赖库版本不同（pandas 1.5 vs 2.0）
  - 系统库缺失（libssl、libffi）
  - 配置文件不同（环境变量、路径）
  - 字符编码不同（UTF-8 vs GBK）
```

#### 虚拟机 vs 容器

```
┌─────────────────────────────────────────────────────────┐
│                    虚拟机 (VM)                            │
├──────────┬──────────┬──────────┬────────────────────────┤
│  App A   │  App B   │  App C   │                        │
│ Bins/Libs│ Bins/Libs│ Bins/Libs│                        │
├──────────┼──────────┼──────────┤      Guest OS          │
│  OS      │  OS      │  OS      │    (每个VM都要装系统)    │
├──────────┴──────────┴──────────┼────────────────────────┤
│          Hypervisor            │                        │
├────────────────────────────────┤      Host OS           │
│          硬件 Infrastructure    │                        │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    容器 (Container)                       │
├──────────┬──────────┬──────────┬────────────────────────┤
│  App A   │  App B   │  App C   │                        │
│ Bins/Libs│ Bins/Libs│ Bins/Libs│   无需Guest OS          │
├──────────┴──────────┴──────────┼────────────────────────┤
│          Docker Engine         │                        │
├────────────────────────────────┤      Host OS           │
│          硬件 Infrastructure    │                        │
└─────────────────────────────────────────────────────────┘
```

| 对比项 | 虚拟机 | 容器 |
|--------|--------|------|
| 启动速度 | 分钟级 | 秒级 |
| 磁盘占用 | GB级（含完整OS） | MB级（共享宿主内核） |
| 运行性能 | 接近原生（有虚拟化损耗） | 接近原生（无虚拟化层） |
| 系统支持 | 每个VM可以是不同OS | 共享宿主内核，必须是Linux |
| 隔离性 | 完全隔离 | 进程级隔离（cgroup + namespace） |
| 适用场景 | 需要完整OS、强隔离 | 微服务、CI/CD、环境标准化 |

#### 容器化在大数据领域的价值

```
大数据开发环境痛点：
  - Hadoop/Spark/MySQL/Redis 组件多，安装配置复杂
  - 团队成员环境不一致，"跑不通"问题频发
  - 版本升级困难，牵一发动全身
  - 新人入职搭环境要一整天

Docker解决方案：
  - 一条命令启动完整开发环境
  - 环境配置写在代码里（Dockerfile），版本可控
  - 团队共享同一镜像，环境100%一致
  - 新人5分钟搞定开发环境
```

### 2.2 Docker核心概念（20分钟）

```
┌───────────────────────────────────────────────────────┐
│               Docker 核心架构                          │
│                                                       │
│   ┌─────────┐    pull/push    ┌──────────────┐        │
│   │  本地    │ ◄─────────────► │   Registry    │       │
│   │  镜像    │                 │  (Docker Hub) │       │
│   │ (Image) │                 └──────────────┘        │
│   └────┬────┘                                          │
│        │ run                                           │
│        ▼                                               │
│   ┌─────────┐                                         │
│   │  容器    │                                         │
│   │(Container)│  ← 镜像的运行实例                       │
│   └─────────┘                                         │
│                                                       │
│   ┌─────────────┐   build   ┌─────────────┐           │
│   │ Dockerfile  │ ────────► │   镜像       │           │
│   │ (构建脚本)   │           │  (Image)     │          │
│   └─────────────┘           └─────────────┘           │
└───────────────────────────────────────────────────────┘

四大核心概念：

1. 镜像 (Image)
   - 只读模板，包含运行应用所需的一切（代码、运行时、库、配置）
   - 分层存储，每一层都是只读的
   - 类比：面向对象中的"类"

2. 容器 (Container)
   - 镜像的运行实例
   - 可写层在镜像之上，删除容器则可写层消失
   - 类比：面向对象中的"对象"

3. 仓库 (Registry)
   - 存放镜像的地方
   - Docker Hub 是最大的公共仓库
   - 企业可搭建私有仓库（Harbor等）

4. Dockerfile
   - 构建镜像的脚本文件
   - 每条指令生成一个镜像层
   - 可版本控制，可复现
```

```bash
# ========== Docker安装（Ubuntu）==========

# 卸载旧版本
sudo apt-get remove docker docker-engine docker.io containerd runc

# 安装依赖
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg

# 添加Docker官方GPG密钥
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 添加仓库
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker Engine
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 验证安装
sudo docker --version
sudo docker compose version

# 将当前用户加入docker组（免sudo）
sudo usermod -aG docker $USER
newgrp docker

# 验证免sudo
docker run hello-world

# ========== Docker安装（Windows/macOS）==========
# 下载安装 Docker Desktop
# https://www.docker.com/products/docker-desktop/
# 安装后重启，终端验证:
docker --version
docker compose version
```

### 2.3 Docker基本操作（40分钟）

```bash
# ========== 镜像操作 ==========

# 搜索镜像
docker search python
docker search mysql
docker search --limit 5 redis

# 拉取镜像
docker pull python:3.11-slim
docker pull mysql:8.0
docker pull redis:7-alpine

# 查看本地镜像
docker images
docker images -a
docker images python

# 镜像标签
docker tag python:3.11-slim mypython:latest

# 删除镜像
docker rmi python:3.11-slim
docker rmi $(docker images -q) -f

# 镜像详情
docker inspect python:3.11-slim

# 镜像历史
docker history python:3.11-slim

# ========== 容器操作 ==========

# 运行容器
docker run hello-world
docker run python:3.11-slim python -c "print('Hello from Docker!')"

# 交互式运行
docker run -it python:3.11-slim /bin/bash
# 进入容器后可以执行命令
# exit 退出

# 后台运行
docker run -d --name my-redis redis:7-alpine

# 常用run参数
docker run -d \
  --name my-mysql \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -e MYSQL_DATABASE=testdb \
  -p 3306:3306 \
  -v mysql_data:/var/lib/mysql \
  mysql:8.0

# 参数说明:
# -d          后台运行（detached）
# --name      容器命名
# -e          设置环境变量
# -p          端口映射 宿主端口:容器端口
# -v          数据卷挂载
# --rm        容器停止后自动删除
# --restart   重启策略（always/on-failure/no）

# 查看容器
docker ps                    # 运行中的容器
docker ps -a                 # 所有容器（含已停止）
docker ps -q                 # 只显示容器ID
docker ps --format "table {{.ID}}\t{{.Names}}\t{{.Status}}"

# 容器生命周期
docker start my-mysql        # 启动已停止的容器
docker stop my-mysql         # 优雅停止（发送SIGTERM）
docker restart my-mysql      # 重启
docker kill my-mysql         # 强制停止（发送SIGKILL）

# 删除容器
docker rm my-mysql           # 删除已停止的容器
docker rm -f my-mysql        # 强制删除（即使运行中）
docker rm $(docker ps -aq)   # 删除所有容器

# 进入运行中的容器
docker exec -it my-mysql /bin/bash
docker exec -it my-mysql mysql -uroot -p123456

# 查看日志
docker logs my-mysql
docker logs -f my-mysql          # 实时跟踪
docker logs --tail 50 my-mysql   # 最后50行
docker logs --since 1h my-mysql  # 最近1小时

# 容器详情
docker inspect my-mysql

# 容器资源占用
docker stats
docker stats my-mysql

# 容器与宿主文件拷贝
docker cp my-mysql:/etc/mysql/my.cnf ./my.cnf
docker cp ./data.csv my-mysql:/tmp/data.csv

# ========== 数据卷操作 ==========

# 创建数据卷
docker volume create mysql_data
docker volume create redis_data

# 查看数据卷
docker volume ls
docker volume inspect mysql_data

# 删除数据卷
docker volume rm mysql_data
docker volume prune

# ========== 网络操作 ==========

# 查看网络
docker network ls

# 创建自定义网络
docker network create bigdata-net

# 容器加入网络
docker run -d --name my-mysql --network bigdata-net mysql:8.0

# 查看网络详情
docker network inspect bigdata-net

# 删除网络
docker network rm bigdata-net
docker network prune
```

### 2.4 Dockerfile编写（45分钟）

#### Dockerfile指令详解

```dockerfile
# ========== Dockerfile 指令速查 ==========

# FROM: 基础镜像（必须是第一条非注释指令）
FROM python:3.11-slim

# WORKDIR: 设置工作目录（不存在会自动创建）
WORKDIR /app

# COPY: 复制文件到镜像中
COPY requirements.txt .
COPY src/ ./src/

# ADD: 类似COPY，但支持URL和自动解压tar
# 一般推荐用COPY，除非需要自动解压
ADD archive.tar.gz /app/

# RUN: 构建时执行命令（生成新的镜像层）
RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    && rm -rf /var/lib/apt/lists/*

RUN pip install --no-cache-dir -r requirements.txt

# ENV: 设置环境变量
ENV PYTHONUNBUFFERED=1
ENV APP_ENV=production

# EXPOSE: 声明容器监听端口（仅文档作用，不实际发布）
EXPOSE 8000

# VOLUME: 声明数据卷挂载点
VOLUME ["/app/data"]

# ARG: 构建时变量（不保留到运行时）
ARG VERSION=1.0

# ENTRYPOINT: 容器入口点（不会被docker run的命令覆盖）
ENTRYPOINT ["python"]

# CMD: 容器启动命令（可被docker run的命令覆盖）
CMD ["app.py"]

# ENTRYPOINT + CMD 组合使用:
# ENTRYPOINT ["python"]
# CMD ["app.py"]
# → 实际执行: python app.py
# → docker run myimage server.py → 实际执行: python server.py
```

#### 实战：编写Python数据处理项目的Dockerfile

```dockerfile
# ========== 项目结构 ==========
# log-analyzer/
# ├── Dockerfile
# ├── requirements.txt
# ├── src/
# │   ├── __init__.py
# │   ├── parser.py
# │   └── analyzer.py
# └── data/
#     └── sample.log

# ---------- requirements.txt ----------
# pandas==2.1.4
# numpy==1.26.2
# matplotlib==3.8.2
# redis==5.0.1
# pymysql==1.1.0
# ----------

# ========== Dockerfile ==========
FROM python:3.11-slim

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY src/ ./src/
COPY data/ ./data/

ENV PYTHONUNBUFFERED=1

VOLUME ["/app/data", "/app/output"]

ENTRYPOINT ["python", "-u"]
CMD ["src/analyzer.py"]
```

```bash
# ========== 构建镜像 ==========
docker build -t log-analyzer:1.0 .
docker build -t log-analyzer:1.0 --no-cache .

# 查看构建的镜像
docker images log-analyzer

# ========== 运行容器 ==========
docker run --rm log-analyzer:1.0

# 覆盖CMD
docker run --rm log-analyzer:1.0 src/parser.py

# 挂载数据目录
docker run --rm \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/output:/app/output \
  log-analyzer:1.0

# ========== 多阶段构建（减小镜像体积）==========
# Dockerfile.multistage
# FROM python:3.11 AS builder
# WORKDIR /build
# COPY requirements.txt .
# RUN pip install --no-cache-dir --prefix=/install -r requirements.txt
#
# FROM python:3.11-slim
# WORKDIR /app
# COPY --from=builder /install /usr/local
# COPY src/ ./src/
# ENTRYPOINT ["python", "-u"]
# CMD ["src/analyzer.py"]

docker build -f Dockerfile.multistage -t log-analyzer:slim .
```

#### Dockerfile最佳实践

```dockerfile
# ========== 最佳实践示例 ==========

# 1. 选择合适的基础镜像
# ❌ FROM ubuntu:latest        (太大，约77MB)
# ❌ FROM python:3.11          (完整版，约1GB)
# ✅ FROM python:3.11-slim     (精简版，约150MB)
# ✅ FROM python:3.11-alpine   (最小，约50MB，但兼容性差)

# 2. 合并RUN指令减少层数
# ❌
# RUN apt-get update
# RUN apt-get install -y gcc
# RUN apt-get install -y make
# ✅
# RUN apt-get update && apt-get install -y --no-install-recommends \
#     gcc \
#     make \
#     && rm -rf /var/lib/apt/lists/*

# 3. 利用构建缓存（把不常变的放前面）
# ✅ 先复制requirements.txt安装依赖
# ✅ 再复制源代码（源代码变化频繁）
# COPY requirements.txt .
# RUN pip install -r requirements.txt
# COPY src/ ./src/

# 4. 使用.dockerignore排除不需要的文件
# ---------- .dockerignore ----------
# __pycache__
# *.pyc
# .git
# .env
# venv
# .venv
# *.md
# .DS_Store
# ----------

# 5. 不要以root用户运行
# RUN useradd -m appuser
# USER appuser
```

### 2.5 Docker Compose（40分钟）

#### 为什么需要Docker Compose

```
单个容器：docker run 就够了

大数据项目通常需要多个服务：
  - MySQL（数据存储）
  - Redis（缓存）
  - Python应用（数据处理）
  - Adminer（数据库管理界面）

手动启动：
  docker run -d --name mysql ...
  docker run -d --name redis ...
  docker run -d --name python ...
  docker run -d --name adminer ...
  → 命令又长又容易出错
  → 容器间网络配置繁琐
  → 启动顺序难以控制

Docker Compose：
  docker compose up -d
  → 一条命令搞定！
```

#### docker-compose.yml 语法

```yaml
# ========== docker-compose.yml 基本结构 ==========

# version: Compose文件版本（已弃用，新版不需要）
services:           # 定义各个服务（容器）
  service-name:
    image: ...      # 使用的镜像
    build: ...      # 或从Dockerfile构建
    container_name: ...
    ports: ...      # 端口映射
    environment: ... # 环境变量
    volumes: ...    # 数据卷
    networks: ...   # 网络
    depends_on: ... # 依赖关系
    restart: ...    # 重启策略

networks:           # 自定义网络
  network-name:
    driver: bridge

volumes:            # 命名数据卷
  volume-name:
```

#### 实战：用Docker Compose启动MySQL + Adminer + Redis

```yaml
# ========== docker-compose.yml ==========
services:
  mysql:
    image: mysql:8.0
    container_name: bigdata-mysql
    environment:
      MYSQL_ROOT_PASSWORD: "123456"
      MYSQL_DATABASE: "testdb"
      MYSQL_USER: "student"
      MYSQL_PASSWORD: "student123"
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    networks:
      - bigdata-net
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: bigdata-redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    networks:
      - bigdata-net
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  adminer:
    image: adminer:latest
    container_name: bigdata-adminer
    ports:
      - "8080:8080"
    networks:
      - bigdata-net
    depends_on:
      mysql:
        condition: service_healthy
    restart: unless-stopped

  python-app:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: bigdata-python
    volumes:
      - ./src:/app/src
      - ./data:/app/data
      - ./output:/app/output
    networks:
      - bigdata-net
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      MYSQL_HOST: "mysql"
      MYSQL_PORT: "3306"
      MYSQL_USER: "student"
      MYSQL_PASSWORD: "student123"
      MYSQL_DATABASE: "testdb"
      REDIS_HOST: "redis"
      REDIS_PORT: "6379"
    stdin_open: true
    tty: true

networks:
  bigdata-net:
    driver: bridge

volumes:
  mysql_data:
  redis_data:
```

# ========== src/db_connector.py ==========
import pymysql
import redis
import os

MYSQL_HOST = os.getenv("MYSQL_HOST", "mysql")
MYSQL_PORT = int(os.getenv("MYSQL_PORT", "3306"))
MYSQL_USER = os.getenv("MYSQL_USER", "student")
MYSQL_PASSWORD = os.getenv("MYSQL_PASSWORD", "student123")
MYSQL_DATABASE = os.getenv("MYSQL_DATABASE", "testdb")

REDIS_HOST = os.getenv("REDIS_HOST", "redis")
REDIS_PORT = int(os.getenv("REDIS_PORT", "6379"))


def get_mysql_connection():
    return pymysql.connect(
        host=MYSQL_HOST,
        port=MYSQL_PORT,
        user=MYSQL_USER,
        password=MYSQL_PASSWORD,
        database=MYSQL_DATABASE,
        charset="utf8mb4",
        cursorclass=pymysql.cursors.DictCursor,
    )


def get_redis_connection():
    return redis.Redis(host=REDIS_HOST, port=REDIS_PORT, decode_responses=True)


def query_users():
    conn = get_mysql_connection()
    try:
        with conn.cursor() as cursor:
            cursor.execute("SELECT * FROM users")
            return cursor.fetchall()
    finally:
        conn.close()


def cache_user_count():
    r = get_redis_connection()
    users = query_users()
    count = len(users)
    r.set("user_count", count, ex=300)
    print(f"已缓存用户数: {count}")
    return count


def get_cached_user_count():
    r = get_redis_connection()
    count = r.get("user_count")
    if count is not None:
        print(f"从缓存获取用户数: {count}")
        return int(count)
    return cache_user_count()


if __name__ == "__main__":
    users = query_users()
    print("用户列表:")
    for u in users:
        print(f"  {u['id']}: {u['name']} - {u['city']}")

    cache_user_count()
    get_cached_user_count()

```python
# ========== src/log_analyzer.py ==========
import sys
import os
from collections import Counter

DATA_DIR = os.getenv("DATA_DIR", "/app/data")
OUTPUT_DIR = os.getenv("OUTPUT_DIR", "/app/output")


def parse_log(filepath):
    results = []
    with open(filepath, "r", encoding="utf-8", errors="ignore") as f:
        for line in f:
            parts = line.strip().split()
            if len(parts) >= 9:
                results.append({
                    "ip": parts[0],
                    "status": parts[8],
                    "url": parts[6] if len(parts) > 6 else "",
                })
    return results


def analyze(records):
    ip_counter = Counter(r["ip"] for r in records)
    status_counter = Counter(r["status"] for r in records)
    url_counter = Counter(r["url"] for r in records)
    return ip_counter, status_counter, url_counter


def write_report(ip_counter, status_counter, url_counter, output_path):
    os.makedirs(os.path.dirname(output_path), exist_ok=True)
    with open(output_path, "w", encoding="utf-8") as f:
        f.write("=== 日志分析报告 ===\n\n")
        f.write("--- TOP 10 IP ---\n")
        for ip, count in ip_counter.most_common(10):
            f.write(f"  {ip}: {count}次\n")
        f.write("\n--- 状态码分布 ---\n")
        for status, count in status_counter.most_common():
            f.write(f"  {status}: {count}次\n")
        f.write("\n--- TOP 10 URL ---\n")
        for url, count in url_counter.most_common(10):
            f.write(f"  {url}: {count}次\n")


def main():
    log_file = sys.argv[1] if len(sys.argv) > 1 else os.path.join(DATA_DIR, "access.log")
    output_file = os.path.join(OUTPUT_DIR, "report.txt")

    print(f"正在分析日志: {log_file}")
    records = parse_log(log_file)
    print(f"共解析 {len(records)} 条记录")

    ip_counter, status_counter, url_counter = analyze(records)
    write_report(ip_counter, status_counter, url_counter, output_file)
    print(f"报告已写入: {output_file}")


if __name__ == "__main__":
    main()
```
