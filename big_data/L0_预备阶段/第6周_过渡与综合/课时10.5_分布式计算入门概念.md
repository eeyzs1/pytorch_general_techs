### 2.1 从单机到分布式：为什么一台机器不够用了？（20分钟）

#### 数据量的爆炸式增长

```
全球数据量增长趋势:

  2010年:   2 ZB   ████
  2015年:  15 ZB   ████████████████████████
  2020年:  64 ZB   ████████████████████████████████████████████████████████████████████████
  2025年: 181 ZB   ██████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████████

  1 ZB = 10^21 字节 = 1,000,000,000,000,000,000,000 字节
  如果一部电影 = 4GB:
  1 ZB ≈ 2500亿部电影
  181 ZB ≈ 45万亿部电影

企业数据规模:
  小公司:   GB ~ TB    (1TB ≈ 1000亿个字符)
  中型公司: TB ~ PB    (1PB = 1024TB)
  大公司:   PB ~ EB    (1EB = 1024PB)
  超大公司: EB+        (微信每天新增PB级数据)
```

#### 摩尔定律的失效

```
摩尔定律（1965年）:
  晶体管数量每18-24个月翻一番 → 单核性能持续提升
  程序员只需等硬件升级，程序自动变快 → "免费的午餐"

2005年前后:
  单核频率触及功耗墙（4-5GHz）
  散热问题无法解决 → 频率不再提升
  转向多核 → 但软件必须改写才能利用多核

2010年至今:
  单机性能提升放缓（每年约3-5%）
  数据量每年增长约25-30%
  → 单机性能增长 << 数据量增长
  → 必须用多台机器协同处理 → 分布式计算
```

### 2.2 分布式系统的核心挑战（25分钟）

#### 挑战1：网络不可靠

```
单机环境:
  函数调用: result = process(data)  ← 100%可靠，要么成功要么崩溃

分布式环境:
  机器A → 网络请求 → 机器B
  可能发生:
    ✅ 请求成功，返回结果
    ⏱️ 请求超时（网络延迟）
    ❌ 请求丢失（网络分区）
    ❌ 响应丢失（B处理完了但A没收到）
    🔄 请求重复到达（网络重传）

  你无法区分:
    - 对方没收到请求
    - 对方收到了但处理时崩溃了
    - 对方处理完了但响应丢了

  → 这就是"网络不可靠"的本质
```

#### 挑战2：时钟不同步

```
单机环境:
  所有操作共享同一个系统时钟
  event1.time = 10:00:00.001
  event2.time = 10:00:00.002
  → 可以确定 event1 先于 event2

分布式环境:
  机器A的时钟: 10:00:00.000
  机器B的时钟: 10:00:00.050  (快了50ms)
  机器C的时钟: 09:59:59.900  (慢了100ms)

  A记录事件: 10:00:00.010
  C记录事件: 09:59:59.950
  → 按时间戳排序: C的事件"先于"A的事件
  → 但实际上A的事件可能先发生！

  真实案例:
    数据库主从复制: 从库的时间戳可能比主库还新
    分布式日志: 不同机器的日志按时间排序可能是错的
```

#### 挑战3：部分故障

```
单机环境:
  要么全好，要么全坏（整机宕机）
  坏了就重启，状态简单

分布式环境:
  100台机器中1台宕机 → 系统还能运行吗？
  - 如果那台机器存着关键数据 → 部分功能不可用
  - 如果那台机器正在执行任务 → 任务失败，需要重试
  - 如果其他机器在等它的结果 → 全部阻塞

  更糟糕的情况:
    机器没宕机，但变慢了（网络抖动、磁盘IO高）
    → 其他机器等它还是跳过它？
    → 这叫"慢节点"问题（straggler）

  部分故障是分布式系统最本质的困难:
    - 你必须为每个操作考虑"如果某一步失败了怎么办"
    - 恢复逻辑往往比正常逻辑更复杂
```

#### 挑战4：一致性

```
单机环境:
  x = 0
  x = x + 1   → x = 1
  x = x + 1   → x = 2
  顺序确定，结果确定

分布式环境:
  数据有3个副本: replica_A, replica_B, replica_C

  客户端1写入: x = 100  → 写入A成功，B成功，C还没收到
  客户端2读取: 从C读取  → 读到旧值 x = 0

  这就是"不一致":
    同一时刻，不同客户端看到不同的数据

  一致性级别:
    强一致性:  写入后，所有后续读取都能看到新值
              （代价: 性能低，延迟高）
    最终一致性: 写入后，过一段时间所有副本才会一致
              （代价: 中间状态可能读到旧值）
    弱一致性:  不保证什么时候一致
              （代价: 数据可能丢失或冲突）

  CAP定理:
    C (Consistency)  一致性
    A (Availability) 可用性
    P (Partition)    分区容错（网络断开）

    网络分区必然发生 → P必须保证
    → 只能在C和A之间二选一:
      CP系统: 保证一致性，牺牲可用性（如ZooKeeper）
      AP系统: 保证可用性，牺牲一致性（如Cassandra）
```

### 2.3 水平扩展 vs 垂直扩展（15分钟）

```
垂直扩展 (Scale-up / 纵向扩展):
  提升单台机器的配置
  4核8G → 8核16G → 32核128G → 128核1T内存

  优点:
    ✅ 不需要改代码
    ✅ 不需要处理分布式问题
    ✅ 运维简单

  缺点:
    ❌ 有上限（最贵的机器也有极限）
    ❌ 单点故障（机器坏了全完）
    ❌ 升级需要停机
    ❌ 越往上越贵（非线性增长）

水平扩展 (Scale-out / 横向扩展):
  增加机器数量
  1台 → 10台 → 100台 → 1000台

  优点:
    ✅ 理论上无限扩展
    ✅ 无单点故障（一台坏了其他继续）
    ✅ 可以在线扩容（不停机）
    ✅ 便宜的普通机器即可

  缺点:
    ❌ 必须处理分布式挑战（网络/时钟/故障/一致性）
    ❌ 代码复杂度大幅增加
    ❌ 运维复杂度增加
    ❌ 数据一致性需要额外保障
```

| 维度 | 垂直扩展 (Scale-up) | 水平扩展 (Scale-out) |
|------|---------------------|---------------------|
| 扩展方式 | 升级单机硬件 | 增加机器数量 |
| 上限 | 有上限（硬件极限） | 理论上无限 |
| 代码改动 | 不需要 | 需要支持分布式 |
| 故障影响 | 单点故障，全部不可用 | 部分故障，系统仍可用 |
| 成本增长 | 非线性（越往上越贵） | 线性（加同配置机器） |
| 典型场景 | 数据库主库、小型应用 | 大数据处理、Web服务 |
| 代表技术 | 大型机、小型机 | Hadoop集群、K8s集群 |

```
大数据领域的选择: 几乎全是 Scale-out

原因:
  1. 数据量太大，单机装不下（PB级数据）
  2. 计算量太大，单机算不过来
  3. 需要高可用，不能有单点故障
  4. 成本控制，1000台普通机器 << 1台超级计算机

Google的教训:
  早期用大型机 → 太贵，扩展困难
  转向普通PC集群 → 便宜，可扩展
  发表GFS/MapReduce/BigTable三篇论文
  → 开源界据此创造了Hadoop
```

### 2.4 分布式计算模型：分而治之（25分钟）

#### MapReduce思想

```
核心思想: 分而治之 (Divide and Conquer)

把大问题拆成小问题 → 并行解决 → 合并结果

生活类比——统计全校学生身高:
  单机: 1个老师量5000人 → 42小时
  MapReduce: 50个老师各量100人 → 50分钟

  Map阶段（分）:  每个老师独立测量，输出<性别, 身高>
  Shuffle阶段（排）: 按性别分组
  Reduce阶段（合）:  分别计算男女平均身高
```

#### 数据并行 vs 任务并行

```
数据并行 (Data Parallelism):
  同样的操作，作用于不同的数据分片
  → MapReduce就是数据并行

  示例: 100GB日志文件分成10份，每台机器处理10GB
  每台机器运行相同的统计代码

  ┌──────────┐  ┌──────────┐  ┌──────────┐
  │ 机器1     │  │ 机器2     │  │ 机器3     │
  │ 数据分片1 │  │ 数据分片2 │  │ 数据分片3 │
  │ 统计代码  │  │ 统计代码  │  │ 统计代码  │
  └─────┬────┘  └─────┬────┘  └─────┬────┘
        │             │             │
        └─────────────┼─────────────┘
                      ▼
                 合并结果

任务并行 (Task Parallelism):
  不同的操作，并行执行
  → 流水线/工作流就是任务并行

  示例: 数据管道
  机器1: 数据采集 → 机器2: 数据清洗 → 机器3: 数据分析

  ┌──────────┐     ┌──────────┐     ┌──────────┐
  │ 机器1     │     │ 机器2     │     │ 机器3     │
  │ 数据采集  │ ──► │ 数据清洗  │ ──► │ 数据分析  │
  └──────────┘     └──────────┘     └──────────┘

大数据系统通常是两者结合:
  Spark: 数据并行（RDD分区）+ 任务并行（DAG调度）
  Flink: 数据并行（算子链并行度）+ 任务并行（算子链式调用）
```

#### MapReduce三阶段详解

```
以WordCount为例——统计文本中每个单词的出现次数:

输入: 3个文本文件

┌─────────────────────────────────────────────────┐
│                 Map阶段（并行）                    │
│                                                   │
│  Mapper1          Mapper2          Mapper3        │
│  "hello world"    "hello spark"   "world of"     │
│  "hello hadoop"   "spark world"   "data"         │
│                                                   │
│  输出:            输出:            输出:           │
│  hello,1          hello,1          world,1        │
│  world,1          spark,1          of,1           │
│  hello,1          spark,1          data,1         │
│  hadoop,1         world,1                         │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│              Shuffle阶段（自动完成）                │
│                                                   │
│  1. Partition: 按Key的hash决定去哪个Reducer       │
│     hash("hello") % 2 = 0 → Reducer 0            │
│     hash("world") % 2 = 1 → Reducer 1            │
│                                                   │
│  2. Sort: 每个Reducer收到的数据按Key排序          │
│                                                   │
│  3. Group: 相同Key的Value合并                     │
│     Reducer 0:                                    │
│       data → [1]                                  │
│       hadoop → [1]                                │
│       hello → [1, 1, 1]                           │
│     Reducer 1:                                    │
│       of → [1]                                    │
│       spark → [1, 1]                              │
│       world → [1, 1, 1]                           │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│               Reduce阶段（并行）                   │
│                                                   │
│  Reducer 0         Reducer 1                     │
│  data → 1          of → 1                        │
│  hadoop → 1        spark → 2                     │
│  hello → 3         world → 3                     │
└─────────────────────────────────────────────────┘
```

### 2.5 用Python模拟分布式计算（30分钟）

#### 实验1：单机 vs multiprocessing并行处理日志

```python
import time
import random
from multiprocessing import Pool, cpu_count
from collections import Counter

def generate_log_file(filepath, n_lines):
    ips = [f"10.{random.randint(0,255)}.{random.randint(0,255)}.{random.randint(1,254)}" for _ in range(500)]
    urls = ["/index.html", "/api/users", "/api/orders", "/login", "/dashboard",
            "/api/products", "/search", "/static/main.js", "/favicon.ico", "/about"]
    statuses = [200]*60 + [301]*5 + [304]*5 + [400]*5 + [404]*10 + [500]*5
    with open(filepath, "w") as f:
        for _ in range(n_lines):
            ip = random.choice(ips)
            url = random.choice(urls)
            status = random.choice(statuses)
            f.write(f'{ip} - - [15/Jan/2024:10:30:00 +0800] "GET {url} HTTP/1.1" {status} 5120\n')

generate_log_file("access.log", 1_000_000)
print(f"已生成100万行日志文件")

def process_chunk(chunk):
    ip_counter = Counter()
    status_counter = Counter()
    for line in chunk:
        parts = line.split()
        if len(parts) >= 9:
            ip_counter[parts[0]] += 1
            status_counter[parts[8]] += 1
    return ip_counter, status_counter

def read_in_chunks(filepath, chunk_size=100000):
    chunk = []
    with open(filepath, "r") as f:
        for line in f:
            chunk.append(line)
            if len(chunk) == chunk_size:
                yield chunk
                chunk = []
    if chunk:
        yield chunk

def single_process():
    start = time.time()
    total_ip = Counter()
    total_status = Counter()
    for chunk in read_in_chunks("access.log", 1_000_000):
        ip_c, st_c = process_chunk(chunk)
        total_ip += ip_c
        total_status += st_c
    elapsed = time.time() - start
    print(f"单进程处理: {elapsed:.3f}秒")
    print(f"  TOP5 IP: {total_ip.most_common(5)}")
    print(f"  状态码: {dict(total_status)}")
    return elapsed

def multi_process():
    start = time.time()
    chunks = list(read_in_chunks("access.log", 100000))
    with Pool(processes=cpu_count()) as pool:
        results = pool.map(process_chunk, chunks)
    total_ip = Counter()
    total_status = Counter()
    for ip_c, st_c in results:
        total_ip += ip_c
        total_status += st_c
    elapsed = time.time() - start
    print(f"多进程处理({cpu_count()}核): {elapsed:.3f}秒")
    print(f"  TOP5 IP: {total_ip.most_common(5)}")
    print(f"  状态码: {dict(total_status)}")
    return elapsed

t1 = single_process()
t2 = multi_process()
print(f"\n加速比: {t1/t2:.2f}x")
print(f"注意: 加速比受CPU核数、IO速度、数据量等因素影响")
print(f"      理想情况下接近核数，但实际通常为核数的60-80%")
```

#### 实验2：用Python模拟MapReduce WordCount

```python
import time
import random
import re
from collections import defaultdict
from multiprocessing import Pool, cpu_count
from functools import reduce as functools_reduce

def generate_text_files(n_files=4, lines_per_file=50000):
    words = ["hello", "world", "spark", "hadoop", "flink", "kafka",
             "data", "stream", "batch", "realtime", "compute", "storage",
             "network", "cluster", "node", "task", "job", "stage",
             "partition", "shuffle", "memory", "disk", "cpu", "gpu"]
    for i in range(n_files):
        with open(f"input_{i}.txt", "w") as f:
            for _ in range(lines_per_file):
                n_words = random.randint(5, 15)
                line = " ".join(random.choices(words, k=n_words))
                f.write(line + "\n")
    print(f"已生成{n_files}个文本文件，每个{lines_per_file}行")

generate_text_files()

def map_phase(filepath):
    word_count = defaultdict(int)
    with open(filepath, "r") as f:
        for line in f:
            for word in line.strip().lower().split():
                word = re.sub(r'[^a-z]', '', word)
                if word:
                    word_count[word] += 1
    return dict(word_count)

def shuffle_phase(map_outputs):
    shuffled = defaultdict(list)
    for output in map_outputs:
        for word, count in output.items():
            shuffled[word].append(count)
    return dict(shuffled)

def reduce_phase(item):
    word, counts = item
    return word, sum(counts)

def run_mapreduce_serial():
    start = time.time()
    import glob
    files = sorted(glob.glob("input_*.txt"))

    map_outputs = []
    for f in files:
        map_outputs.append(map_phase(f))

    shuffled = shuffle_phase(map_outputs)

    results = {}
    for word, counts in shuffled.items():
        results[word] = sum(counts)

    elapsed = time.time() - start
    print(f"串行MapReduce: {elapsed:.3f}秒")
    top5 = sorted(results.items(), key=lambda x: -x[1])[:5]
    print(f"  TOP5单词: {top5}")
    return elapsed, results

def run_mapreduce_parallel():
    start = time.time()
    import glob
    files = sorted(glob.glob("input_*.txt"))

    with Pool(processes=cpu_count()) as pool:
        map_outputs = pool.map(map_phase, files)

    shuffled = shuffle_phase(map_outputs)

    with Pool(processes=cpu_count()) as pool:
        reduce_results = pool.map(reduce_phase, shuffled.items())

    results = dict(reduce_results)

    elapsed = time.time() - start
    print(f"并行MapReduce({cpu_count()}核): {elapsed:.3f}秒")
    top5 = sorted(results.items(), key=lambda x: -x[1])[:5]
    print(f"  TOP5单词: {top5}")
    return elapsed, results

t1, r1 = run_mapreduce_serial()
t2, r2 = run_mapreduce_parallel()

print(f"\n加速比: {t1/t2:.2f}x")
print(f"结果一致性: {'通过' if r1 == r2 else '不一致'}")

print("\n=== MapReduce模拟流程回顾 ===")
print("1. Map阶段:   每个文件独立统计单词频率 → 并行")
print("2. Shuffle阶段: 按单词分组，收集所有Map的输出")
print("3. Reduce阶段: 对每个单词的计数求和 → 并行")
print("核心: Map和Reduce可以并行，Shuffle需要汇总数据")
```

```
单机思维 → 分布式思维:

1. 数据在哪里？
   单机: 数据在本地磁盘，直接读取
   分布式: 数据分布在多台机器，需要考虑数据本地性
           → "把计算移到数据旁边，而不是把数据移到计算旁边"

2. 程序怎么运行？
   单机: python script.py
   分布式: 提交作业到集群，由调度器分配资源
           → 需要考虑: 资源申请、任务调度、依赖管理

3. 出错了怎么办？
   单机: try/except，重试
   分布式: 必须假设"随时会出错"
           → 需要考虑: 重试策略、幂等性、检查点

4. 结果怎么收集？
   单机: 直接返回
   分布式: 多台机器的结果需要汇总
           → 需要考虑: Shuffle、数据倾斜、内存溢出

5. 状态怎么管理？
   单机: 全局变量
   分布式: 全局变量无法跨机器共享
           → 需要考虑: 无状态设计、外部存储

思维转换清单:

  │ 原子操作          │ 分布式锁/最终一致性           │

  │ 调试: print      │ 调试: 日志聚合+链路追踪       │
  │ 部署: scp+run    │ 部署: 调度器+容器编排         │
  └──────────────────┴──────────────────────────────┘
```

### 练习1：用Python multiprocessing并行处理CSV文件（15分钟）

```python
import csv
import time
import random
from multiprocessing import Pool, cpu_count
from collections import Counter

def generate_csv(filepath, n_rows):
    cities = ["北京", "上海", "广州", "深圳", "杭州", "成都", "武汉", "南京"]
    categories = ["电子产品", "服装", "食品", "家居", "图书"]
    with open(filepath, "w", newline="", encoding="utf-8") as f:
        writer = csv.writer(f)
        writer.writerow(["order_id", "city", "category", "amount"])
        for i in range(n_rows):
            city = random.choice(cities)
            category = random.choice(categories)
            amount = round(random.uniform(10, 5000), 2)
            writer.writerow([f"ORD{i:08d}", city, category, amount])

generate_csv("orders.csv", 1_000_000)
print("已生成100万行CSV")

def process_csv_chunk(args):
    filepath, start_row, end_row = args
    city_sales = Counter()
    category_sales = Counter()
    with open(filepath, "r", encoding="utf-8") as f:
        reader = csv.reader(f)
        next(reader)
        for i, row in enumerate(reader):
            if i < start_row:
                continue
            if i >= end_row:
                break
            if len(row) >= 4:
                city_sales[row[1]] += float(row[3])
                category_sales[row[2]] += float(row[3])
    return city_sales, category_sales

def parallel_csv():
    n_rows = 1_000_000
    n_workers = cpu_count()
    chunk_size = n_rows // n_workers
    tasks = []
    for i in range(n_workers):
        start = i * chunk_size
        end = (i + 1) * chunk_size if i < n_workers - 1 else n_rows
        tasks.append(("orders.csv", start, end))

    start = time.time()
    with Pool(n_workers) as pool:
        results = pool.map(process_csv_chunk, tasks)

    total_city = Counter()
    total_category = Counter()
    for city_c, cat_c in results:
        total_city += city_c
        total_category += cat_c

    elapsed = time.time() - start
    print(f"并行处理({n_workers}核): {elapsed:.3f}秒")
    print(f"城市销售额TOP5: {total_city.most_common(5)}")
    print(f"品类销售额TOP5: {total_category.most_common(5)}")

parallel_csv()
```

### 练习2：手动模拟MapReduce的map和reduce阶段（15分钟）

```python
from collections import defaultdict

sample_data = [
    "apple banana apple cherry",
    "banana cherry date apple",
    "cherry date elderberry fig",
    "apple banana cherry date",
    "elderberry fig apple banana",
]

print("=== 手动模拟MapReduce ===\n")

print("Step 1: Map阶段（每行独立处理）")
map_outputs = []
for i, line in enumerate(sample_data):
    words = line.split()
    pairs = [(word, 1) for word in words]
    map_outputs.extend(pairs)
    print(f"  Mapper {i}: {line}")
    print(f"    输出: {pairs}")

print(f"\nMap阶段总输出: {len(map_outputs)}个键值对")

print("\nStep 2: Shuffle阶段（按Key分组）")
shuffled = defaultdict(list)
for key, value in map_outputs:
    shuffled[key].append(value)
for key in sorted(shuffled.keys()):
    print(f"  {key}: {shuffled[key]}")

print("\nStep 3: Reduce阶段（对每组求和）")
results = {}
for key, values in shuffled.items():
    results[key] = sum(values)
    print(f"  {key}: {values} → sum = {results[key]}")

print("\n最终结果:")
for word in sorted(results.keys()):
    print(f"  {word}: {results[word]}")

print("\n=== 验证: 用Counter直接统计 ===")
from collections import Counter
all_words = []
for line in sample_data:
    all_words.extend(line.split())
print(dict(Counter(all_words)))
```

## 五、关键概念速查

| 概念 | 含义 | 类比 |
|------|------|------|
| 分布式系统 | 多台独立机器协同工作 | 多人协作完成大项目 |
| Scale-up | 升级单机配置 | 把一个人训练成超人 |
| Scale-out | 增加机器数量 | 招更多人一起干 |
| Map | 并行处理数据分片 | 每人各统计一部分 |
| Shuffle | 按Key重新分组 | 把相同类别的材料归到一起 |
| Reduce | 对分组数据聚合 | 每人汇总一类结果 |
| 数据并行 | 同操作不同数据 | 100人各搬100块砖 |
| 任务并行 | 不同操作并行执行 | 有人搬砖有人砌墙 |
| CAP定理 | 一致性/可用性/分区容错三选二 | 不可能三角 |
| 最终一致性 | 过一段时间数据一致 | 信息传播需要时间 |
| 数据本地性 | 计算靠近数据 | 就近取材 |
