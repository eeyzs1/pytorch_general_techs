## 二、数据质量六维度模型

### 2.1 六维度定义

```
数据质量六维度金字塔:

              ┌──────────┐
              │  有效性   │  ← 数据格式是否符合业务规则
              └────┬─────┘
           ┌──────┴──────┐
           │   唯一性    │  ← 是否存在重复记录
           └──────┬──────┘
        ┌─────────┴─────────┐
        │     一致性        │  ← 跨表/跨系统数据是否一致
        └─────────┬─────────┘
     ┌────────────┴────────────┐
     │        准确性           │  ← 数据是否反映真实世界
     └────────────┬────────────┘
  ┌───────────────┴───────────────┐
  │           及时性              │  ← 数据是否在SLA内到达
  └───────────────┬───────────────┘
┌─────────────────┴─────────────────┐
│            完整性                 │  ← 必填字段是否为空
└───────────────────────────────────┘
```

### 2.2 各维度详解与示例

| 维度 | 定义 | 典型检查规则 | 适用层级 |
|------|------|-------------|----------|
| 完整性 | 必填字段无缺失 | 非空检查、行数不为零 | ODS/DWD |
| 准确性 | 数据值与真实值一致 | 金额范围检查、枚举值校验 | DWD/DWS |
| 一致性 | 跨表/跨系统数据一致 | 外键引用完整性、主从表金额一致 | DWD/ADS |
| 及时性 | 数据在SLA内到达 | ETL完成时间检查、数据新鲜度 | ODS |
| 唯一性 | 无重复记录 | 主键唯一性、业务去重 | ODS/DWD |
| 有效性 | 数据格式符合规则 | 正则匹配、值域范围、日期格式 | ODS/DWD |

### 2.3 按数仓层级设计质量规则

```
ODS层质量规则（偏原始性检查）:
  ├── 完整性: 核心字段非空率 > 99%
  ├── 唯一性: 主键无重复
  ├── 及时性: 数据在每日06:00前到达
  └── 行数波动: 当日行数与7日均值偏差 < 50%

DWD层质量规则（偏清洗后一致性）:
  ├── 外键一致性: order_id在订单主表中存在
  ├── 枚举值检查: order_status ∈ {已创建,已支付,已完成,已取消}
  ├── 时间范围: create_time在合理区间内
  └── 格式有效性: 手机号/身份证号格式校验

DWS层质量规则（偏指标合理性）:
  ├── 指标波动: 环比变化幅度 < 50%
  ├── 空值率: 关键指标空值率 < 5%
  ├── 汇总一致性: DWS汇总金额 = DWD明细金额之和
  └── 业务合理性: 客单价在 [10, 50000] 区间

ADS层质量规则（偏业务规则）:
  ├── GMV一致性: ADS层GMV = DWD层订单金额汇总
  ├── 留存率范围: 0% ≤ 留存率 ≤ 100%
  └── TOP N完整性: TOP10榜单无空值
```

## 三、Great Expectations实战

### 3.1 核心概念

```
Great Expectations 工作流:

┌─────────────┐    ┌──────────────────┐    ┌──────────────┐    ┌────────────┐
│  DataSource  │───→│  ExpectationSuite │───→│  Checkpoint  │───→│  Data Docs │
│  (数据源)    │    │  (期望规则集)     │    │  (检查点)    │    │  (文档报告)│
└─────────────┘    └──────────────────┘    └──────┬───────┘    └────────────┘
                                                  │
                                           ┌──────▼───────┐
                                           │  Validation  │
                                           │  Result      │
                                           │  (验证结果)  │
                                           └──────────────┘

关键概念:
  DataSource:   数据连接配置（数据库、文件等）
  Expectation:  单条数据质量规则（如"列值非空"）
  ExpectationSuite: 一组Expectation的集合，绑定到某张表
  Checkpoint:   运行验证的入口，将数据+规则+动作组合
  Data Docs:    自动生成的HTML质量报告
  ValidationResult: 单次验证的详细结果
```

### 3.3 为ODS层定义Expectation Suite

```python
import great_expectations as gx
from great_expectations.core import ExpectationSuite

context = gx.get_context()

suite_ods_order_info = context.add_expectation_suite(
    expectation_suite_name="ods_order_info_suite"
)

suite = context.get_expectation_suite("ods_order_info_suite")

suite.add_expectation(
    gx.expectations.ExpectColumnValuesToNotBeNull(column="order_id")
)
suite.add_expectation(
    gx.expectations.ExpectColumnValuesToNotBeNull(column="user_id")
)
suite.add_expectation(
    gx.expectations.ExpectColumnValuesToNotBeNull(column="total_amount")
)
suite.add_expectation(
    gx.expectations.ExpectColumnValuesToBeUnique(column="order_id")
)
suite.add_expectation(
    gx.expectations.ExpectColumnValuesToBeInSet(
        column="order_status",
        value_set=["已创建", "已支付", "已完成", "已取消", "已退款"]
    )
)
suite.add_expectation(
    gx.expectations.ExpectColumnValuesToBeBetween(
        column="total_amount",
        min_value=0,
        max_value=1000000
    )
)
suite.add_expectation(
    gx.expectations.ExpectTableRowCountToBeBetween(
        min_value=100,
        max_value=5000000
    )
)
suite.add_expectation(
    gx.expectations.ExpectColumnValuesToMatchRegex(
        column="create_time",
        regex=r"^\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}$"
    )
)
suite.add_expectation(
    gx.expectations.ExpectColumnValuesToNotBeNull(column="create_time", mostly=0.99)
)
suite.add_expectation(
    gx.expectations.ExpectColumnPairValuesAToBeGreaterThanB(
        column_A="total_amount",
        column_B="payment_amount"
    )
)

suite.add_expectation(
    gx.expectations.ExpectColumnValuesToNotBeNull(column="shipping_province", mostly=0.95)

context.add_or_update_expectation_suite(expectation_suite=suite)
```

### 3.5 运行Checkpoint

```python
import great_expectations as gx
from great_expectations.checkpoint import Checkpoint

context = gx.get_context()

datasource = context.sources.add_spark("spark_datasource")
data_asset = datasource.add_dataframe_asset(name="ods_order_info")

batch_request = data_asset.build_batch_request(
    dataframe=spark.sql("SELECT * FROM ods.ods_order_info WHERE dt='2024-01-15'")
)

checkpoint = Checkpoint(
    name="ods_order_info_checkpoint",
    data_context=context,
    validations=[
        {
            "batch_request": batch_request,
            "expectation_suite_name": "ods_order_info_suite"
        }
    ]
)

context.add_or_update_checkpoint(checkpoint=checkpoint)
result = checkpoint.run()

print(f"验证结果: 成功={result['success']}")
print(f"统计: 总计={result['run_results']}")
```

### 3.6 与Airflow集成

```python
from airflow import DAG
from airflow.operators.python import PythonOperator
from datetime import datetime, timedelta
import great_expectations as gx

default_args = {
    'owner': 'data-quality',
    'depends_on_past': False,
    'retries': 1,
    'retry_delay': timedelta(minutes=5),
}

dag = DAG(
    'data_quality_check',
    default_args=default_args,
    schedule_interval='0 7 * * *',
    start_date=datetime(2024, 1, 1),
    catchup=False,
)

def run_ge_checkpoint(**context):
    ds = context['ds']
    context_gx = gx.get_context()
    checkpoint = context_gx.get_checkpoint("ods_order_info_checkpoint")
    result = checkpoint.run()
    if not result['success']:
        raise ValueError(f"数据质量检查失败: {ds}")

def send_alert(**context):
    import requests
    webhook_url = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY"
    ds = context['ds']
    payload = {
        "msgtype": "markdown",
        "markdown": {
            "content": f"**数据质量告警**\n> 日期: {ds}\n> 检查: ODS层质量检查失败\n> 请及时排查"
        }
    }
    requests.post(webhook_url, json=payload)

dq_check_task = PythonOperator(
    task_id='run_dq_check',
    python_callable=run_ge_checkpoint,
    provide_context=True,
    on_failure_callback=send_alert,
    dag=dag,
)
```

## 四、Deequ实战（Amazon开源）

### 4.1 Deequ核心概念

```
Deequ 工作流（基于Spark）:

┌──────────────┐    ┌───────────────────┐    ┌──────────────────┐
│  Spark       │───→│  VerificationSuite│───→│  VerificationResult│
│  DataFrame   │    │  (验证套件)       │    │  (验证结果)       │
└──────────────┘    └───────────────────┘    └──────────────────┘
                            │
                     ┌──────┴──────┐
                     │  Analyzer   │
                     │  (指标分析) │
                     └──────┬──────┘
                            │
                     ┌──────▼──────┐
                     │  Metrics    │
                     │  Repository │
                     │  (指标存储) │
                     └─────────────┘

特点:
  - 基于Spark，天然支持大规模数据
  - 声明式API定义约束
  - 支持指标计算和异常检测
  - 可将指标存入MySQL供趋势分析
```

### 4.2 PyDeequ数据质量指标计算

```python
from pyspark.sql import SparkSession
import pydeequ
from pydeequ.analyzers import *
from pydeequ.checks import *
from pydeequ.verification import *

spark = SparkSession.builder \
    .appName("DeequDQ") \
    .config("spark.jars.packages", pydeequ.deequ_maven_coord) \
    .getOrCreate()

df = spark.sql("SELECT * FROM ods.ods_order_info WHERE dt='2024-01-15'")

analysisResult = AnalysisRunner(spark) \
    .onData(df) \
    .addAnalyzer(Size()) \
    .addAnalyzer(Completeness("order_id")) \
    .addAnalyzer(Completeness("user_id")) \
    .addAnalyzer(Completeness("total_amount")) \
    .addAnalyzer(Uniqueness(["order_id"])) \
    .addAnalyzer(Mean("total_amount")) \
    .addAnalyzer.StandardDeviation("total_amount")) \
    .addAnalyzer(Minimum("total_amount")) \
    .addAnalyzer(Maximum("total_amount")) \
    .addAnalyzer(ApproxCountDistinct("user_id")) \
    .run()

analysisResult_df = AnalyzerContext.successMetricsAsDataFrame(spark, analysisResult)
analysisResult_df.show(truncate=False)
```

### 4.3 Deequ约束验证

```python
from pydeequ.checks import *
from pydeequ.verification import *

check = Check(spark, CheckLevel.Error, "ODS Order Info Check")

checkResult = VerificationSuite(spark) \
    .onData(df) \
    .addCheck(
        check \
        .isComplete("order_id") \
        .isUnique("order_id") \
        .isComplete("user_id") \
        .isComplete("total_amount") \
        .isNonNegative("total_amount") \
        .isLessThan("total_amount", lambda x: x < 1000000) \
        .isContainedIn("order_status", ["已创建", "已支付", "已完成", "已取消", "已退款"]) \
        .hasCompleteness("shipping_province", lambda x: x >= 0.95) \
        .hasUniqueness(["order_id"], lambda x: x == 1.0) \
        .hasSize(lambda x: x >= 100)
    ) \
    .run()

checkResult_df = VerificationResult.checkResultsAsDataFrame(spark, checkResult)
checkResult_df.select("check", "check_level", "check_status", "constraint", "constraint_status", "constraint_message").show(truncate=False)
```

### 4.4 Deequ异常检测

```python
from pydeequ.anomaly_detection import *

anomalyDetectionResult = VerificationSuite(spark) \
    .onData(df) \
    .addAnomalyCheck(
        RelativeRateOfChangeStrategy(maxRateIncrease=1.5),
        Size()
    ) \
    .run()

anomalyResult_df = VerificationResult.checkResultsAsDataFrame(spark, anomalyDetectionResult)
anomalyResult_df.show(truncate=False)
```

## 五、Soda实战

### 5.1 核心概念

```
Soda 工作流:

┌──────────────────┐    ┌──────────────┐    ┌──────────────────┐
│  checks.yml      │───→│  soda scan   │───→│  Soda Cloud      │
│  (YAML声明规则)  │    │  (扫描执行)  │    │  (云端可视化)    │
└──────────────────┘    └──────┬───────┘    └──────────────────┘
                               │
                        ┌──────▼───────┐
                        │  CLI输出     │
                        │  (终端报告)  │
                        └──────────────┘

特点:
  - YAML声明式定义检查规则，无需写代码
  - 与dbt深度集成，可在dbt模型上定义检查
  - 支持多种数据源: Hive, Spark, PostgreSQL, MySQL, Snowflake
  - Soda Cloud提供可视化质量Dashboard
```

### 5.2 YAML声明式数据检查

```yaml
# checks_ods_order_info.yml
data_source: hive_warehouse
checks for ods_order_info:
  - schema:
      name: 订单主表Schema检查
      fail:
        when required_column_missing:
          - order_id
          - user_id
          - total_amount
          - order_status
          - create_time
  - row_count > 0:
      name: 订单主表行数不为空
  - row_count < 5000000:
      name: 订单主表行数上限
  - missing_count(order_id) = 0:
      name: order_id无缺失
  - missing_count(user_id) = 0:
      name: user_id无缺失
  - duplicate_count(order_id) = 0:
      name: order_id无重复
  - values_in(order_status) in ["已创建", "已支付", "已完成", "已取消", "已退款"]:
      name: order_status枚举值校验
  - min(total_amount) >= 0:
      name: total_amount非负
  - max(total_amount) < 1000000:
      name: total_amount上限
  - invalid_count(create_time) = 0:
      name: create_time格式有效
      valid format: "%Y-%m-%d %H:%M:%S"
```

## 六、元数据管理平台

### 6.1 三大平台对比

| 维度 | DataHub | Apache Atlas | OpenMetadata |
|------|---------|-------------|-------------|
| 开发方 | LinkedIn/Acorn | Apache | OpenMetadata社区 |
| 语言 | Java + Python | Java | Java |
| 架构 | Kafka + ES + MySQL | HBase + Solr | ES + MySQL |
| 元数据模型 | 通用且灵活 | 强绑定Hadoop生态 | 通用，支持多种源 |
| 血缘支持 | 强（支持Airflow/Spark/Flink） | 强（Hive/Spark为主） | 强（多源支持） |
| 部署难度 | 中等（Docker一键部署） | 高（依赖HBase/ZK） | 低（Docker一键部署） |
| UI体验 | 现代化，搜索强大 | 较旧，功能齐全 | 现代化，直观 |
| 社区活跃度 | 非常活跃 | 一般 | 活跃 |
| 适用场景 | 通用企业级 | Hadoop生态深度用户 | 中小团队快速上手 |

### 7.1 OpenLineage标准

```
OpenLineage 核心模型:

┌──────────────────────────────────────────────────┐
│                  RunEvent                        │
│                                                  │
│  ┌────────────┐  ┌────────────┐  ┌───────────┐ │
│  │   Run      │  │    Job     │  │  Inputs   │ │
│  │  (运行实例)│  │  (作业定义)│  │ (输入数据集)│ │
│  └────────────┘  └────────────┘  └───────────┘ │
│                                       │         │
│                                ┌──────▼──────┐  │
│                                │  Outputs    │  │
│                                │ (输出数据集) │  │
│                                └─────────────┘  │
└──────────────────────────────────────────────────┘

关键概念:
  Run:      一次作业执行实例，包含开始/结束时间和状态
  Job:      作业定义（如Airflow Task、Spark Job）
  Dataset:  输入/输出数据集（如Hive表、Kafka Topic）
  Facet:    附加元信息（如SQL语句、Schema变更）
```

### 7.3 配置Airflow推送血缘到Marquez

```ini
# airflow.cfg 追加配置

[openlineage_airflow]
```

```python
# airflow_openlineage_dag.py
from airflow import DAG
from airflow.operators.bash import BashOperator
from airflow.providers.apache.hive.operators.hive import HiveOperator
from datetime import datetime, timedelta

default_args = {
    'owner': 'data-engineering',
    'depends_on_past': False,
    'retries': 2,
    'retry_delay': timedelta(minutes=5),
}

with DAG(
    'ods_to_dwd_order',
    default_args=default_args,
    schedule_interval='0 6 * * *',
    start_date=datetime(2024, 1, 1),
    catchup=False,
) as dag:

    load_ods = HiveOperator(
        task_id='load_ods_order_info',
        hql="""
            INSERT OVERWRITE TABLE ods.ods_order_info PARTITION (dt='{{ ds }}')
            SELECT order_id, user_id, order_status, total_amount,
                   payment_amount, shipping_province, shipping_city,
                   create_time, pay_time, finish_time
            FROM staging.stg_order_info
            WHERE dt='{{ ds }}'
        """,
    )

    build_dwd = HiveOperator(
        task_id='build_dwd_order_detail',
        hql="""
            INSERT OVERWRITE TABLE dwd.dwd_order_detail PARTITION (dt='{{ ds }}')
            SELECT
                d.detail_id, d.order_id, o.user_id,
                u.username AS user_name, u.gender AS user_gender,
                u.user_level, u.province AS user_province, u.city AS user_city,
                d.sku_id, d.sku_name, s.category_id, s.category_name, s.brand,
                d.order_price, d.sku_num, o.total_amount, o.payment_amount,
                o.order_status, d.create_time,
                YEAR(d.create_time) AS order_year,
                MONTH(d.create_time) AS order_month,
                DAY(d.create_time) AS order_day,
                WEEKOFYEAR(d.create_time) AS order_week,
                HOUR(d.create_time) AS order_hour
            FROM ods.ods_order_detail d
            JOIN ods.ods_order_info o ON d.order_id = o.order_id
            JOIN ods.ods_user_info u ON o.user_id = u.user_id
            JOIN ods.ods_sku_info s ON d.sku_id = s.sku_id
            WHERE d.dt='{{ ds }}' AND o.dt='{{ ds }}'
              AND u.dt='{{ ds }}' AND s.dt='{{ ds }}'
        """,
    )

    load_ods >> build_dwd
```
